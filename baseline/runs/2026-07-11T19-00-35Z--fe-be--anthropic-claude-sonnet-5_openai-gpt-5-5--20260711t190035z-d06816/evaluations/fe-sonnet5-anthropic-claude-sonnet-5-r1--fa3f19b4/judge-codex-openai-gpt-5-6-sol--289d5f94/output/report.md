# Frontend evaluation — id-fa3f19b4

## Evaluation basis

- The complete seed-to-submission delta is one new file: `submission/prototype.html` (3,351 lines). The specifications, SDK references, and nine reference screenshots are unchanged.
- Static validation: the extracted script passes `node --check`. A lightweight DOM/localStorage runtime harness executed the submission's actual seed, router, renderers, dispatch mutations, serialization, reload, reset, and malformed-state paths.
- Runtime route coverage: Home, project dock, all six project tools, message/to-do/card/document details, Activity, Calendar, Reports, Everything, Search, Adminland, Pings, Assignments, Drafts, and My Boosts all produced non-error render output. A missing item produced the intended Not Found surface.
- Mutation coverage: tested to-do completion, bookmark, message creation, comment, boost, card move, trash/restore, save/reload, invalid JSON, valid but incomplete stored state, and reset.
- Pixel references were inspected directly. Browser screenshot automation could not be used because the environment denied local socket binding and Chromium's Mach rendezvous, and the installed browser connector exposed no browser. Visual scores therefore combine direct reference inspection with the implemented geometry, tokens, responsive rules, and rendered markup.

## Verified behavior distinctions

- **Present and working:** broad route navigation, deterministic Basecamp sample graph, comments/boosts, completion, card movement, create flows through their working entry points, filtering/search, keyboard shortcuts, and most mutations.
- **Working and persistent:** tested to-do, bookmark, created message/comment/boost, moved card, and trash/restore state.
- **Present but stubbed:** invite/profile/support, project people/notifications, several list/filter/sort controls, file folder/upload/cloud link and bulk actions, chat attachments/emoji/voice, schedule event creation, and subscription controls.
- **Present but unwired:** the primary Message Board `New message` button and Docs & Files `Document` menu item.
- **Persistent with material integrity gaps:** ordinary changes survive reload, while reset is incomplete and syntactically valid malformed state can replace core collections and trigger the error boundary.

## Bottom line

This is an unusually broad, product-aware single-file prototype with a strong deterministic Basecamp model and many real workflows. Its strongest areas are product fidelity and surface breadth. Its ceiling is set by conspicuous stubs/unwired entry points, brittle global rerendering and focus behavior, incomplete reset/malformed-state handling, and limited mobile/accessibility hardening.
